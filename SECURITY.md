## 报告安全漏洞

我们非常重视项目的安全性。如果你发现了任何安全漏洞，请按照以下步骤报告：

### 📧 如何报告

1. **不要**公开报告漏洞（不要创建公开的 Issue）
2. 通过以下方式之一联系我们：
   - GitHub Security Advisory（推荐）
   - 发送邮件到：[你的邮箱地址]

### 📋 报告内容

请在报告中包含以下信息：

- 漏洞的描述
- 复现步骤
- 影响范围
- 可能的修复方案（如果有）
- 你的联系方式（可选）

### ⏱️ 响应时间

我们承诺：
- 在 **48 小时内**确认收到你的报告
- 在 **7 天内**提供初步评估
- 及时向你更新修复进度

### 🎁 致谢

如果你报告的漏洞被确认并修复，我们会：
- 在发布说明中感谢你（如果你同意）
- 如果你愿意，可以将你添加到贡献者列表中

## 安全最佳实践

### 对于用户

- 保持你的依赖项是最新的
- 定期检查安全更新
- 不要在代码中提交敏感信息（API Key、密码等）
- 使用 `.gitignore` 排除敏感文件

### 对于开发者

- 在提交代码前进行安全审查
- 使用静态代码分析工具
- 编写安全的代码（输入验证、错误处理等）
- 定期更新依赖项
