## Reporting a Vulnerability

We take the security of our project very seriously. If you discover any security vulnerabilities, please report them following the steps below:

### 📧 How to Report

1. **Do NOT** report vulnerabilities publicly (don't create a public Issue)
2. Contact us via one of the following methods:
   - GitHub Security Advisory (recommended)
   - Email to: [your email address]

### 📋 What to Include

Please include the following information in your report:

- Description of the vulnerability
- Steps to reproduce
- Impact scope
- Possible fix (if any)
- Your contact information (optional)

### ⏱️ Response Time

We commit to:
- Acknowledge receipt of your report within **48 hours**
- Provide an initial assessment within **7 days**
- Keep you updated on the fix progress in a timely manner

### 🎁 Acknowledgment

If the vulnerability you reported is confirmed and fixed, we will:
- Thank you in the release notes (if you agree)
- Add you to the contributors list if you wish

## Security Best Practices

### For Users

- Keep your dependencies up to date
- Regularly check for security updates
- Don't commit sensitive information (API keys, passwords, etc.) in code
- Use `.gitignore` to exclude sensitive files

### For Developers

- Conduct security reviews before committing code
- Use static code analysis tools
- Write secure code (input validation, error handling, etc.)
- Update dependencies regularly
